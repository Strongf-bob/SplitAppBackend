# Прозрачный тачбар PWA — дизайн

> Исторический дизайн. PWA-контур удалён 2026-07-14; этот документ не предназначен для дальнейшей реализации.

## Цель

Заменить перегруженный нижний тачбар, стилизованный под Liquid Glass, на спокойную полупрозрачную рейку. Навигация остаётся прежней: пять вкладок, безопасные зоны iOS и немедленное переключение экранов не меняются.

## Выбранное визуальное направление

Выбран вариант A из визуального сравнения: единая мягкая прозрачная рейка поверх синего экрана.

- Контейнер имеет один полупрозрачный светлый фон, `backdrop-filter: blur(...)`, тонкую светлую границу и одну мягкую внешнюю тень.
- Контейнер не использует декоративные слои Liquid Glass: нет зерна, радужных/радиальных бликов, `mix-blend-mode`, псевдо-элементов для сияния или нескольких вложенных фильтров.
- Активная вкладка получает простую светлую полупрозрачную плашку. Неактивные элементы не получают фон при hover, который менял бы иерархию интерфейса.
- Высота, отступ от safe area, ширина, размеры области нажатия и короткая анимация `transform` сохраняются. Анимация не затрагивает layout и отключается для `prefers-reduced-motion` существующим глобальным правилом.

## Иконки

- Для вкладок «Главная», «Друзья», «События» и «Профиль» используются системные Unicode emoji, отрисованные системным emoji-шрифтом устройства; PNG-иконки этих четырёх вкладок больше не подключаются.
- Центральная вкладка «Сплитик» сохраняет текущий фирменный PNG `nav-add.png`, чтобы не потерять узнаваемость ассистента.
- Все кнопки сохраняют доступное текстовое имя через видимую подпись или скрытую подпись для центральной вкладки. Декоративная иконка помечается `aria-hidden`.

## Производительность и совместимость

- На тачбаре остаётся один слой `backdrop-filter`; активная плашка не создаёт собственного blur-слоя.
- Fallback для браузеров без `backdrop-filter` — непрозрачнее, но визуально такой же спокойный сине-серый фон и та же граница.
- Не меняются маршруты, состояние `activeView`, обработчик `onNavigate`, порядок вкладок и разметка safe-area. Это исключает влияние дизайна на навигационную логику.

## Изменяемые файлы

- `web/src/app/page.tsx`: модель пунктов навигации и разметка нижней навигации.
- `web/src/app/globals.css`: стили тачбара, активного пункта и fallback.
- `web/tests/pwa-ui-contract.test.mjs`: контрактные проверки новой структуры, системных emoji, центрального PNG, safe area и облегчённого CSS.
- `web/public/sw.js`: версия cache name, так как изменяется оболочка установленной PWA.

## Проверка

1. Контрактный тест PWA сначала фиксирует новую модель и падает на старой реализации.
2. После реализации выполняются `node --test tests/*.test.mjs`, `npm run typecheck`, `npm run build`, `make test`, `make lint` и `make format-check`.
3. В локальном PWA проверяются клики по всем пяти вкладкам, отсутствие горизонтальной прокрутки и читаемость бара на синем и белом участках экранов.
