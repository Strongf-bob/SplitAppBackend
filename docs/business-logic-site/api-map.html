API-карта SplitApp Backend

Эта страница синхронизирована с текущими routers: Auth, Home, Users, Friends, Events, Receipts, Payments, Disputes, Audit, Reports, Splitik и Health. Для машинного контракта источником остается openapi.yaml.

Runtime, публичная страница и health

GET/

Статичная публичная страница о нативном iOS-приложении.

GET/assets/landing/*

Статические assets публичной страницы.

GET/api/ping

Легкий liveness check.

GET/api/health/db

Проверка MongoDB connection.

GET/api/metrics

Prometheus metrics; в production должен быть защищен токеном/прокси.

Ротация refresh token и выдача нового access token.

GET/api/home/summary

Confirmed, pending и disputed money buckets для текущего пользователя.

GET/api/users

Пагинированные visible users: actor и участники общих событий.

GET/api/users/search

Opt-in поиск по handle/name для discovery_enabled=true.

Счетчики open/closed events и outstanding money buckets.

PATCH/api/users/me

Имя, email, avatar, public handle, discovery и visibility payment phone.

POST/api/users/me/contacts/import

Owner-scoped import контактов после явного выбора пользователем.

GET/api/users/me/contacts

Пагинированный список импортированных контактов и matched users.

Пагинация и idempotency

List endpoints возвращают общий envelope:

{
  "items": [],
  "limit": 50,
  "offset": 0,
  "total": 128
}

Формат используется для users, contacts, events, receipts, share reviews, payments, payment requests, disputes, audit events и friends.

Money-changing create endpoints требуют Idempotency-Key:

POST /api/events/{id}/receipts
POST /api/events/{id}/payments
POST /api/events/{id}/payment-requests
POST /api/payment-requests/{id}/mark-paid

Повтор сетевого запроса не должен создавать второй финансовый документ.
